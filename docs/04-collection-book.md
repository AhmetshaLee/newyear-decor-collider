# Документ 4: Книга Коллекций (CollectionBook) и Инвентарь

## 1. Концепция Книги Коллекций и Инвентаря

- **Правило фиксации коллекции:** Как только предмет сгенерирован в Коллайдере или получен в Календаре, его уникальный идентификатор (`id`) мгновенно заносится в архив открытых наград (массив `unlockedCollectionIds` в `localStorage`).
- **Механика разбития в Инвентаре:** Экран Инвентаря рендерит сетку NxN физически находящихся у игрока предметов. Если игрок уничтожает предмет кнопкой «Разбить», предмет удаляется из массива `inventory`, а баланс `userShards` пополняется на 50% от стоимости крафта этого типа. Из массива книги коллекций `unlockedCollectionIds` предмет **НЕ удаляется** и остается там открытым навсегда.
- **Генерация наград (Рандом):** При крафте система фильтрует базу данных `decorations-registry.json` по выставленным на панели Коллайдера параметрам (Альбом, Уровень, Тип) и выдает случайную игрушку. Если антиповторитель находится в режиме `USE SHARDS`, из пула выбора предварительно исключаются все `id`, которые уже присутствуют в массиве `unlockedCollectionIds`, а доплата списывается из баланса осколков вместе с основной стоимостью крафта.
- **Контент реестра украшений:** Записи `decorations-registry` являются контентом в рамках уже утвержденной модели. Для них можно создавать `id`, названия и художественные описания, если задача явно касается наполнения реестра. При этом нельзя без отдельной спецификации добавлять новые альбомы, уровни, типы, валюты, правила выпадения или механики коллекции.

## 2. Структура Экрана Книги Коллекций (Роут `/collection`)

Экран представляет собой интерактивный альбом с четырьмя перелистываемыми страницами, соответствующими новогодним коллекциям:

1. `classic` — Новогодняя классика
2. `fairytale` — Рождественская сказка
3. `oriental` — Восточный календарь
4. `magic` — Зимнее чудо / Магия.

### 2.1 Монитор прогресса (Статистика)

Вверху каждой страницы отображается цифровое табло со счетчиками:

1.  **Общий прогресс:** [Всего открыто уникальных игрушек во всех коллекциях] / [Общее количество существующих игрушек в JSON].
2.  **Прогресс текущей страницы:** [Открыто в этой коллекции] / [Всего игрушек в этой коллекции].
3.  **Процент заполнения страницы** (например, `45%`).

### 2.2 Сетка-матрица коллекции (Альбомный лист)

Отображает сетку всех существующих игрушек данной тематики из JSON-реестра.

- **Неоткрытый слот:** Если `id` предмета отсутствует в массиве `unlockedCollectionIds`, ячейка должна визуально выглядеть как пустой/заблокированный призрачный слот.
- **Открытый слот:** Если `id` присутствует в массиве, ячейка отображает полноценную цветную иконку, а при наведении (или тапе на мобильных устройствах) выводится всплывающее окно (Tooltip или Modal) с художественным русским описанием, названием предмета и его уровнем (I-V).
