# Документ 3: Коллайдер Крафта

Этот документ описывает бизнес-логику крафта, состояние панели, инварианты элементов управления и расчет стоимости. Правила отображения холста, фиксированного фрейма, панели масштаба и вписывания прибора в экран описаны отдельно в `docs/03a-collider-canvas.md`.

---

## 1. Техническое описание Информационного дисплея (Монитор)

ЖК-экран зеленого свечения выводит строки текста, синхронизированные со стейтом:

- Строка 1: `Available shards: [userShards]`
- Строка 2: `DECORATION PROJECT`
- Строка 3: `Level: [Random / I / II / III / IV / V]` (зависит от ротора уровня)
- Строка 4: `Album: [Random / Classic / Fairytale / Oriental / Magic]` (зависит от ротора альбома)
- Строка 5: `Type: [Random / Top / Lights / Toys / Floor]` (зависит от выбранного типа украшения под ель)
- Строка 6: `Anti-repeat: [Off / Use N shards]` (зависит от состояния антиповторителя; точное значение `N` задается балансом)

> [!IMPORTANT]
> Поле `Object` удалено из панели и рецепта крафта. Экран больше не разделяет "основной тип" и "подтип": остается только один параметр `Type`.

> [!IMPORTANT]
> Режим больших украшений удален из прототипа коллайдера. Отдельного тумблера `LARGE DECORATION`, состояния `STANDARD / LARGE` и модификатора цены за большое украшение больше нет.

## 2. Логика элементов управления и Взаимосвязи (Инварианты)

### 2.1 Блок «ALBUM»

> [!NOTE]
> В ранних черновиках этот параметр назывался `Collection` как прямой перевод "коллекции", но в пользовательском интерфейсе и документации корректное название — **Альбом**.

Ротор на 5 дискретных положений. Выбор меток вокруг сразу вращает ручку на нужный угол, клик по центру ручки переключает положения циклически:

1. `random` — Случайный альбом (дефолт, модификатор цены = 0).
2. `classic` — Новогодняя классика (модификатор = +150).
3. `fairytale` — Рождественская сказка (модификатор = +150).
4. `oriental` — Восточный календарь (модификатор = +150).
5. `magic` — Зимнее чудо / Магия (модификатор = +150).

### 2.2 Блок «LEVEL»

Ротор на 6 дискретных положений, управляющий уровнем создаваемой игрушки:

1. `random` — Случайный уровень (дефолт, модификатор цены = 0).
2. `lvl_1` (+50 осколков) | 3. `lvl_2` (+100) | 4. `lvl_3` (+200) | 5. `lvl_4` (+400) | 6. `lvl_5` (+800).

### 2.3 Блок выбора типа «DECORATION TYPE»

> [!IMPORTANT]
> Матрица 4x4 удалена из текущей модели крафта. Коллайдер больше не поддерживает столбцы `FEAST`, `INSTALLATION`, `YARD` и отдельный параметр `Object`.

Блок состоит из 5 кнопок:

1. `random` — Случайный тип (дефолт, модификатор цены = 0).
2. `top` — Верхушка.
3. `lights` — Гирлянды.
4. `toys` — Навесные игрушки.
5. `floor` — Нижние игрушки.

**Инвариант:** в блоке активна только одна кнопка. Выбор любой конкретной кнопки гасит «RANDOM», нажатие на «RANDOM» гасит выбранный конкретный тип.

### 2.4 Модуль «АНТИПОВТОРИТЕЛЬ»

- **Элемент:** Тумблер на 2 положения: `OFF` / `USE SHARDS`.
- **Состояние `OFF`:** Антиповторитель не влияет на пул наград. В рецепте допускаются повторы уже открытых игрушек.
- **Состояние `USE SHARDS`:** При успешном крафте к итоговой цене добавляется фиксированная доплата в осколках. Точное значение доплаты задается балансом отдельно (ориентир — около `500` осколков, но это не финальная константа). Система гарантирует выпадение игрушки, которой еще нет в Книге коллекций.
- **Упрощение модели:** Отдельная сущность зарядов антиповторителя удалена. Поля `chargesAvailable`, дисплей зарядов, списание зарядов и блокировка тумблера из-за нулевого количества зарядов больше не используются.

### 2.5 Кнопки панелей

Под информационным дисплеем расположены две служебные кнопки:

- `[ I ]` открывает нижнюю панель Инвентаря и устанавливает в URL `?panel=inventory`;
- `[ C ]` открывает правую панель Книги коллекций и устанавливает в URL `?panel=collection`.

Панели отображаются поверх Коллайдера. Их открытие и закрытие не изменяет временное состояние настроек рецепта.

## 3. Математическая модель расчета стоимости и кнопка «СТАРТ»

`Итоговая цена = База (100) + Модификатор Альбома + Модификатор Уровня + Модификатор Типа + Доплата Антиповторителя`

- Выбран конкретный альбом (позиции 2-5): `+150`.
- Выбран конкретный уровень (позиции 2-6): `+50` / `+100` / `+200` / `+400` / `+800`.
- Выбран конкретный тип украшения под ель вместо «RANDOM»: `+250`.
- Антиповторитель выключен (`OFF`): `+0`.
- Антиповторитель в режиме `USE SHARDS`: `+N` осколков, где `N` — балансируемая фиксированная доплата.

### Условия блокировки кнопки «СТАРТ» (`disabled`):

1. `userShards` меньше Итоговой цены.

При успешном нажатии: ресурсы списываются, генерируется случайный объект украшения из пула, отфильтрованного по рецепту (альбом, уровень, тип), добавляется в массив `inventory` и заносится в Книгу коллекций.
