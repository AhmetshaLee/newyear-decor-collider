# Документ 3: Коллайдер Крафта

Этот документ описывает бизнес-логику крафта, состояние панели, инварианты элементов управления и расчет стоимости. Правила отображения холста, фиксированного фрейма, панели масштаба и вписывания прибора в экран описаны отдельно в `docs/03a-collider-canvas.md`.

---

## 1. Техническое описание Информационного дисплея (Монитор)

ЖК-экран зеленого свечения выводит строки текста, синхронизированные со стейтом:

- Строка 1: `Available shards: [userShards]`
- Строка 2: `[STANDARD / LARGE] DECORATION PROJECT` (зависит от тумблера)
- Строка 3: `Level: [Random / I / II / III / IV / V]` (зависит от ротора уровня)
- Строка 4: `Collection: [Random / Classic / Fairytale / Oriental / Magic]` (зависит от ротора коллекции)
- Строка 5: `Type: [Random / Название выбранного столбца матрицы]`
- Строка 6: `Object: [Random / Название выбранного подтипа строки]`
- Строка 7: `Duplication chance: [Yes (если антиповторитель OFF) / No (если антиповторитель ON)]`

## 2. Логика элементов управления и Взаимосвязи (Инварианты)

### 2.1 Блок «COLLECTION»

Ротор на 5 дискретных положений. Выбор меток вокруг сразу вращает ручку на нужный угол, клик по центру ручки переключает положения циклически:

1. `random` — Случайная коллекция (дефолт, модификатор цены = 0).
2. `classic` — Новогодняя классика (модификатор = +150).
3. `fairytale` — Рождественская сказка (модификатор = +150).
4. `oriental` — Восточный календарь (модификатор = +150).
5. `magic` — Зимнее чудо / Магия (модификатор = +150).

### 2.2 Блок «LEVEL»

Ротор на 6 дискретных положений, управляющий уровнем создаваемой игрушки:

1. `random` — Случайный уровень (дефолт, модификатор цены = 0).
2. `lvl_1` (+50 осколков) | 3. `lvl_2` (+100) | 4. `lvl_3` (+200) | 5. `lvl_4` (+400) | 6. `lvl_5` (+800).

### 2.3 Блок «LARGE DECORATION»

Вертикальный тумблер (флажок ON/OFF). При переводе в положение `ON` центральная матрица типов (17 кнопок) и ротор LEVEL принудительно гаснут, блокируются (`disabled`) и сбрасываются в режим `random`. Большие украшения не имеют уровней и обычных подтипов. При выключении тумблера блокировка со всей панели снимается.

### 2.4 Центральная матрица типов «DECORATION TYPE»

Состоит из 17 кнопок: кнопка «RANDOM» (отдельная кнопка) и сетка зависимых кнопок 4x4 (4 столбца на 4 строки).

- **Столбцы (Основные типы):** `CHRISTMAS TREE` (Ель), `FEAST` (Ярмарка), `INSTALLATION` (Инсталляция), `YARD` (Двор).
- **Строки (Уникальные подтипы для каждого столбца):**
  - _CHRISTMAS TREE:_ Верхушка (`top`), Гирлянды (`lights`), Навесные игрушки (`toys`), Нижние игрушки (`floor`).
  - _FEAST:_ Палатки (`tents`), Иллюминация (`illumination`), Карусель (`attractions`), Фотозоны (`street_decor`).
  - _INSTALLATION:_ Снеговик (`center_figure`), Лед (`ice_fence`), Олени (`light_sculptures`), Скамейки (`environment`).
  - _YARD:_ Ограждение, Окна, Фасад, Фонари.
- **Инвариант:** В сетке 4х4 горит только одна ячейка. Выбор любой ячейки гасит кнопку «RANDOM», нажатие на «RANDOM» гасит активную ячейку в сетке.

### 2.5 Модуль «АНТИПОВТОРИТЕЛЬ»

- **Элемент:** Тумблер на 2 положения: `OFF` / `ON`. Ниже находится цифровой дисплей количества зарядов.
- **Связанное правило:** Включение режима `ON` возможно только если `chargesAvailable > 0`. Если зарядов 0, тумблер аппаратно заблокирован в положении `OFF`. При успешном крафте с включенным антиповторителем списывается 1 заряд, а система гарантирует выпадение игрушки, которой еще нет в Книге коллекций. Стоимость в осколках при этом НЕ растет.

## 3. Математическая модель расчета стоимости и кнопка «СТАРТ»

`Итоговая цена = База (100) + Модификатор Коллекции + Модификатор Уровня + Модификатор Типа + Модификатор Большого Украшения`

- Выбрана конкретная коллекция (позиции 2-5): `+150`.
- Выбран конкретный уровень (позиции 2-6): `+50` / `+100` / `+200` / `+400` / `+800`.
- Выбрана конкретная ячейка в сетке 4x4 вместо «RANDOM»: `+250`.
- Тумблер «Большое украшение» включен: `+500`.

### Условия блокировки кнопки «СТАРТ» (`disabled`):

1. `userShards` меньше Итоговой цены.
2. Антиповторитель включен (`ON`), но баланс зарядов равен `0`.

При успешном нажатии: ресурсы списываются, генерируется случайный объект украшения из пула, отфильтрованного по рецепту, добавляется в массив `inventory` и заносится в Книгу коллекций.
