## 1. Технические улучшения и Инструментарий

### 1.1. Режим Разработчика (Debug Panel)

Для комфортного тестирования механик (особенно календаря и тайм-локов) необходима скрытая панель разработчика.

- **Активация:** 5 быстрых кликов по зеленому ЖК-монитору коллайдера.
- **Функции:** Кнопки `+1000 Shards`, `Add Charge`, `+1 Day to Calendar` (эмуляция смены суток), `Reset State`. Панель должна рендериться поверх игры (абсолютное позиционирование).

### 1.2. Версионирование сохранений (State Migrations)

Заложить в структуру `localStorage` поле `version: 1`. В будущем, при добавлении новых коллекций или изменении структуры данных, это позволит легко написать функцию-мигратор (перевод конфига с `v1` на `v2` при загрузке приложения) без сброса прогресса игроков.

### 1.3. Тактильный звук и Вибрация (Web Audio / Haptic)

Чтобы аналоговая панель ощущалась "живой", прикрутить физический отклик на взаимодействие с элементами управления:

- Использовать **Web Audio API** (Oscillator) для генерации синтезированных коротких щелчков при переключении тумблеров и треска искр при крафте (это не требует загрузки mp3-файлов).
- Использовать **Vibration API** (`navigator.vibrate([10, 30, 10])`) для тактильного отклика (Haptic Feedback) на мобильных устройствах при повороте роторов.

## 2. Геймдизайн и Механики

### 2.1. Награды за сбор коллекций (Completion Bonus)

Добавить смысл в полное закрытие страниц Книги коллекций. Когда прогресс страницы достигает 100% (собраны все игрушки конкретной тематики), игрок получает перманентный бонус:

- Либо скидка 10-15% на крафт игрушек этой коллекции.
- Либо уникальный косметический предмет (например, золотая рамка вокруг аватара или другой цвет свечения монитора).

### 2.2. Механика «Перегрузки Коллайдера» (Risk/Crit)

Если игрок выставляет на панели самые дорогие параметры (Level V + конкретная коллекция + конкретный тип + Большое украшение), есть шанс спровоцировать системный сбой:

- **Критический успех:** Коллайдер выдает сразу две разные игрушки вместо одной.
- **Предохранитель:** На мониторе загорается `SYSTEM OVERLOAD: SHARDS REFUNDED`, крафт отменяется, осколки возвращаются.
  Это добавит элемент казино-азарта при крупных тратах.

## 3. Интерфейс, Анимации и UX

### 3.1. Аналоговая тактильность (CSS Animations)

Все интерактивные элементы должны физически реагировать на клик:

- Тумблеры должны щелкать с эффектом пружины (`cubic-bezier` с overshoot).
- Роторы имеют инерцию при вращении.
- Кнопки матрицы 4x4 должны визуально "вдавливаться" в панель при активации.

### 3.2. Эффект запуска системы (Boot Sequence)

При первой загрузке приложения (монтировании React-дерева) делать короткую анимацию включения старого ЭЛТ-монитора:

- Сначала яркая вспышка, затем сужение в зеленую полосу (как в старых телевизорах).
- На ЖК-экране быстро пробегают строки инициализации: `Loading shards... Scanning inventory... READY`.

### 3.3. Визуализация Антиповторителя

Сделать тумблер антиповторителя самым визуально-выразительным элементом на панели:

- Если режим `ON` и есть заряды: неоновая колба рядом с ним пульсирует ярким цветом (`filter: drop-shadow`).
- Если заряды закончились (`0`): колба мигает красным, издает звук разряженной батареи и аппаратно блокирует тумблер, переводя его в `OFF`.

## 4. Математика и Баланс

### 4.1. Взвешенный рандом (Weighted Random Selection)

Если один из параметров крафта не зафиксирован (выбран режим `Random`), алгоритм выборки из JSON-реестра не должен быть равновероятным:

- Обычные предметы (Level I-II) имеют высокий вес (например, 50-70).
- Редкие предметы (Level IV-V) имеют низкий вес (например, 5-10).
  Это предотвратит поломку экономики игры, когда игрок за базовые 100 осколков постоянно достает самые дорогие и редкие вещи.
